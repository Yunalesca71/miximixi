## Context

Miximixi是一个3D餐厅选择应用，使用React + Three.js构建。当前布局在不同屏幕尺寸下存在以下问题：
- 大屏幕上元素分散，标题左对齐，按钮位置偏右上
- 小屏幕上标题、副标题和按钮布局不够紧凑
- 卡牌位置与上方元素间距不合理
- 缺乏统一的响应式断点系统

当前技术栈：React 19、Vite、Three.js、GSAP动画、CSS模块

## Goals / Non-Goals

**Goals:**
- 建立统一的响应式布局系统，支持手机、平板、桌面三种主要设备
- 确保标题(图标+文字)和副标题在所有屏幕尺寸下横向居中
- 优化按钮布局，小屏幕下单行显示，大屏幕下合理分布
- 改进卡牌垂直定位，确保与上方元素有适当间距
- 保证所有内容在一页内完整显示，无需滚动

**Non-Goals:**
- 不改变3D卡牌的核心功能和交互逻辑
- 不修改GSAP动画效果
- 不改变应用的核心业务逻辑

## Decisions

### 1. 响应式断点系统
**决策**: 采用标准的三断点系统
- Mobile: < 768px
- Tablet: 768px - 1024px  
- Desktop: > 1024px

**理由**: 这是业界标准，覆盖了主要设备类型，便于维护和扩展

### 2. 布局架构重构
**决策**: 使用Flexbox布局替代当前的absolute定位
- Header使用flex布局实现居中
- Control Panel根据屏幕尺寸调整定位策略
- Main Content使用flex布局优化垂直空间分配

**理由**: Flexbox提供更好的响应式支持，减少复杂的媒体查询

### 3. 响应式检测方案
**决策**: 采用react-responsive库进行响应式检测
- 使用useMediaQuery Hook检测屏幕尺寸
- 在组件内部根据屏幕尺寸动态渲染不同布局
- 替代现有的window.innerWidth监听方式

**理由**: 
- 与React组件紧密集成，代码更清晰
- 减少手动resize事件监听的复杂性
- 提供声明式的响应式编程方式
- 与现有Three.js和GSAP系统无冲突

### 4. 尺寸适配策略
**决策**: 结合react-responsive和CSS变量
- 使用react-responsive控制布局结构
- 使用CSS变量管理具体尺寸值
- 字体使用rem单位，基于根字体大小
- 间距使用rem和百分比组合

**理由**: 提供更好的可扩展性和一致性，同时保持样式灵活性

### 5. 卡牌容器定位
**决策**: 使用视口单位优化垂直定位
- 大屏幕：padding-top: 20vh
- 小屏幕：padding-top: 15vh
- 动态调整卡牌容器的垂直位置

**理由**: 确保卡牌在不同屏幕尺寸下都有合适的显示位置

## Risks / Trade-offs

**[风险]** 现有样式覆盖可能导致样式冲突 → **缓解**: 使用CSS模块化命名空间，逐步迁移样式

**[风险]** GSAP动画可能与新布局冲突 → **缓解**: 保持现有的DOM结构，只调整定位和尺寸

**[风险]** react-responsive可能影响性能 → **缓解**: 使用React.memo优化组件，避免不必要的重渲染

**[风险]** 不同设备的字体渲染差异 → **缓解**: 使用系统字体栈，添加字体回退方案

**[权衡]** 布局复杂度增加 vs 用户体验提升 → 接受适度的复杂度提升以换取更好的用户体验

**[权衡]** 引入新依赖 vs 维护成本 → react-responsive是轻量级库，长期维护成本更低

## Migration Plan

1. **依赖安装**: 安装react-responsive库
2. **准备阶段**: 备份现有CSS文件，创建新的响应式样式文件
3. **重构Header**: 修改App.jsx，使用react-responsive实现标题居中布局
4. **重构Control Panel**: 调整RestaurantSelector组件，使用useMediaQuery控制按钮布局
5. **优化Main Content**: 重新设计卡牌容器定位和间距
6. **测试验证**: 在不同设备和屏幕尺寸下测试布局效果
7. **清理优化**: 移除冗余样式和手动resize监听，优化性能

**回滚策略**: 保留原始CSS文件的备份，如出现问题可快速回滚
